\section*{Prodotti attesi}
% Personalizzare definendo i prodotti attesi (facoltativo)
Al termine del periodo di stage, lo studente dovrà consegnare i seguenti prodotti:
\begin{enumerate}
    \item Modulo applicativo del gestionale interno \\
    Implementazione di un modulo funzionante composto da un backend NestJS 11 (su Node.js 24 LTS) con API REST documentate, persistenza su database PostgreSQL 18 e interfaccia utente realizzata in Next.js 16 con Tailwind CSS 4, integrata al backend tramite le API sviluppate.

    \item Sistema di autenticazione e autorizzazione \\
    Implementazione di un meccanismo di autenticazione basato su JWT, con gestione utenti, ruoli e protezione degli endpoint tramite middleware, guard e validazioni.

    \item Documentazione tecnica del modulo sviluppato \\
    Documentazione che descriva l'architettura adottata, le scelte progettuali, la struttura delle API e le istruzioni di utilizzo del modulo, prodotta in formato leggibile e versionata insieme al codice.

    \item Relazione finale di stage \\
    Relazione scritta che illustri il percorso formativo svolto, le tecnologie apprese, le attività realizzate, le difficoltà incontrate e le soluzioni adottate, con un bilancio degli obiettivi raggiunti rispetto a quelli pianificati.

    \item Presentazione finale \\
    Demo del modulo sviluppato e presentazione del lavoro svolto al tutor aziendale e agli stakeholders coinvolti.
\end{enumerate}

Nel caso in cui lo studente, completate le attività obbligatorie, abbia ancora tempo a disposizione, potrà cimentarsi nella realizzazione degli obiettivi desiderabili e facoltativi, in particolare estendendo il modulo verso un'architettura SaaS multitenant, introducendo meccanismi di caching con Redis 8 o approfondendo i flussi di deploy sull'infrastruttura cloud aziendale.
